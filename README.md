# Speech-to-Text API Benchmark

Hands-on benchmark comparing four ASR (Automatic Speech Recognition) APIs across 8 audio samples and 320 API calls.

**Providers tested:** Cohere Transcribe, OpenAI Whisper, Deepgram Nova-2, AssemblyAI

Built for the [AI News Home](https://ainewshome.com) article on Cohere's new transcription model.

## Results

| # | Provider | WER | Median Speed |
|---|----------|-----|-------------|
| 1 | Cohere Transcribe | 3.2% | 1.42s |
| 2 | AssemblyAI | 3.3% | 3.94s |
| 3 | OpenAI Whisper | 3.6% | 2.41s |
| 4 | Deepgram Nova-2 | 4.6% | 1.34s |

WER = word error rate (duration-weighted, lower is better). Speed = median API response time including network latency.

Full results with per-sample breakdowns in `reports/results_final.json`.

## Audio samples

| Sample | Category | Duration | Reference |
|--------|----------|----------|-----------|
| Clean English | Baseline | 10.4s | Consensus |
| Harvard Sentences | Baseline | 33.6s | Ground truth |
| Noisy (15 dB SNR) | Noise | 33.6s | Ground truth |
| Noisy (5 dB SNR) | Noise | 33.6s | Ground truth |
| Indian-Accented English | Accent | 24.5s | Ground truth |
| French-Accented English | Accent | 33.2s | Ground truth |
| Gettysburg Address | Long-form | 125.3s | Ground truth |
| MLK Speech | Historical | 13.0s | Consensus |

Noisy samples are synthetically generated by mixing pink noise (Voss-McCartney algorithm) into the Harvard Sentences audio at calibrated SNR levels (verified at 15.0 dB and 5.0 dB).

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Set your API keys:

```bash
export CO_API_KEY=your_cohere_key
export OPENAI_API_KEY=your_openai_key
export DEEPGRAM_API_KEY=your_deepgram_key
export ASSEMBLYAI_API_KEY=your_assemblyai_key
```

## Usage

```bash
# Full benchmark (10 runs per sample per provider)
python run.py --runs 10

# Quick test (3 runs)
python run.py --runs 3

# Test specific providers
python run.py --providers cohere openai

# Dry run (validate setup, no API calls)
python run.py --dry-run
```

The script downloads audio samples automatically, runs the benchmark, and generates an HTML report in `reports/`.

## Methodology

- 10 runs per sample per provider (320 total API calls)
- WER computed with [jiwer](https://github.com/jitsi/jiwer) (Levenshtein word edit distance)
- Duration-weighted WER: longer samples have proportionally more influence on averages
- Timing = wall-clock API call including network latency
- AssemblyAI uses an async API (upload + poll), inflating its wall-clock time
- Ground truth transcripts used where available; consensus reference (majority vote) otherwise
- Single test location; results may vary by region and time of day
